This crate wraps LibJIT in an idiomatic style. For example, here's a quick example which makes a multiply function using LibJIT:

#![feature(phase)]
extern crate jit;
#[phase(plugin)]
extern crate jit_macros;
use jit::{Context, get, Type};
fn main() {
    // make a new context to make functions on
    let mut ctx = Context::new();
    jit_func!(ctx, func, fn mul(x: int, y: int) -> int {
        let result = func.insn_mul(x, y);
        func.insn_return(&result);
    }, |mul| {
        assert_eq!(mul((4, 5)), 20);
    });
}

Type

An object that represents a native system type. Each Type represents a basic system type, be it a primitive, a struct, a union, a pointer, or a function signature. The library uses this information to lay out values in memory. Types are not attached to a context so they are reference-counted by LibJIT, so internally they are represented as Rc<TypeData>.

Value

Values form the backbone of the storage system in libjit. Every value in the system, be it a constant, a local variable, or a temporary result, is represented by an object of type Value. The JIT then allocates registers or memory locations to the values as appropriate.
